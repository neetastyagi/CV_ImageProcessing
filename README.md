# CV_ImageProcessing

A collection of Python scripts and image-processing functions for an image-processing assignment (PS1). The repository contains implementations of basic image manipulation operations (channel extraction and swapping, copy/paste regions, statistics, normalization, shifting, difference, noise addition) and utilities to build and visualize hybrid images.

### Stack
- **Language(s):** Python 3 (primary)
- **Framework / runtime:** Plain Python scripts (cv2 / OpenCV)
- **Notable libraries:** OpenCV (cv2), NumPy

## How it's organized
```
README.md                <-- this file
Readme.md                <-- short assignment note
ps1.py                   <-- library of image-processing functions (core of the project)
experiment.py            <-- example script that runs the assignment pipeline and writes outputs
Try.py                   <-- small interactive script demonstrating channel extraction
try2.py                  <-- experiment for masking / copy-paste examples
ps1-*.png                <-- output/example images generated by experiment.py (results)
dog.bmp, cat.bmp         <-- source images used for hybrid image example
cat.bmp / dog.bmp        <-- example bitmaps for hybrid images
southafricaflagface.png  <-- example input image used in Try.py
__pycache__/             <-- python cache (ignored for most purposes)
```

How it fits together:
- ps1.py implements functions used by the assignment: channel extraction (extract_red/green/blue), swap_green_blue, copy/paste helpers (copy_paste_middle, copy_paste_middle_circle), image statistics and normalization (image_stats, center_and_normalize), geometric transforms (shift_image_left), difference_image, add_noise, and hybrid-image helpers (build_hybrid_image, vis_hybrid_image).
- experiment.py is the driver that loads sample images, runs the functions from ps1.py, asserts image size constraints, and writes example outputs (ps1-*.png, hybrid_image_scales.png). It is the recommended starting point to reproduce the outputs.
- Try.py and try2.py are smaller exploratory scripts that demonstrate channel display, masking, and copy/paste experiments.

## How to run it
Prerequisites: Python 3.8+ (or 3.x), pip. The project uses OpenCV (cv2) and NumPy.

Example setup and run commands:

```bash
# 1) create a virtual environment (optional but recommended)
python3 -m venv .venv
source .venv/bin/activate

# 2) install dependencies
pip install --upgrade pip
pip install numpy opencv-python

# 3) run the example pipeline
python experiment.py
```

What experiment.py does (short): it loads two images for the color-plane and replacement exercises, performs channel swaps and extractions, copies regions (square and circular), computes statistics and normalizations, shifts images and computes differences, adds Gaussian noise to channels, and builds a hybrid image from dog.bmp and cat.bmp. The script writes its outputs as ps1-*.png and hybrid_image_scales.png in the repository root.

Notes and potential pitfalls:
- experiment.py expects certain images to exist in the repository root (for example ps1-1-a-1.png, ps1-1-a-2.png, dog.bmp, cat.bmp). If you replace or rename images, update the filenames at the top of experiment.py.
- Some helper scripts (Try.py) reference `fruit.png` or `southafricaflagface.png` — ensure those input files are present if you run them, or edit the filename.
- Functions in ps1.py often return float arrays (for normalization) — when writing to disk with cv2.imwrite you may want to cast to uint8 or normalize appropriately.
